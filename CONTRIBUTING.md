# Контрибьютинг

## Принципы (из ТЗ)

1. **Excel — источник истины.** Перед кодом расчётного блока разбери формулы листа, выпиши алгоритм псевдокодом, потом пиши Python.
2. **Сначала тесты, потом фичи.** Для каждого расчётного модуля сперва тест на контрольных примерах (вход → эталон из Excel), затем реализация.
3. **Работа по этапам** (см. README / docs/roadmap.md). Не реализуем все методики сразу.
4. **Не угадывать формулы.** Непонятно → запись в [OPEN_QUESTIONS.md](OPEN_QUESTIONS.md).
5. **Не хардкодить** тексты и цены — конфиг/БД/`core/content`.

## Перед коммитом

```bash
ruff check .
black --check .
pytest
```

## Коммиты

Conventional Commits: английский префикс + **русское** описание, инфинитив, ≤72 символа, без точки.

```
feat(payment): добавить оплату Telegram Stars
fix(biorhythm): исправить расчёт критических дней
test(psychomatrix): добавить сверку с Excel
docs: уточнить open questions по анкете
```

Полная шпаргалка — `dev-base/templates/conventional-commits.md`.

## PR / Issues

На русском, по шаблонам в `.github/`. Секреты в коммит не попадают — `.env` в `.gitignore`, в репозитории только `.env.example`.
