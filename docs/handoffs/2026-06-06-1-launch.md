# Handoff для Cowork — запуск бота нумерологической матрицы

**Статус кода:** всё готово и в `main`. Расчётное ядро сверено с Excel-эталоном
(нулевой допуск), бот собран, Docker-образ проверен. Нужны действия, которые я
(Claude) сделать не могу: регистрация бота в Telegram и деплой на сервер.

## Что сделать

### 1. Создать бота в @BotFather (браузер/Telegram)
- `/newbot` → имя и username (например `…matrix_bot`).
- Сохранить **BOT_TOKEN**.
- (Опц.) задать описание/about/картинку; команды: `start`, `cancel`, `admin`.
- Telegram Stars включать НЕ нужно — это встроенная валюта `XTR`, provider_token не требуется.
- Узнать **Telegram ID** администратора (например через @userinfobot) — для `/admin` и `/refund`.

### 2. Деплой на сервер с Docker (см. docs/deploy.md)
```bash
git clone https://github.com/m34959203/numerology-bot.git
cd numerology-bot
cp .env.example .env
# заполнить в .env: BOT_TOKEN, ADMIN_IDS=<id админа>, POSTGRES_PASSWORD=<свой пароль>
# REPORT_FORMAT=pdf (по умолчанию)
docker compose up -d --build
docker compose logs -f bot   # дождаться "запуск бота"
```
Контейнер сам применит миграции (alembic) и засеет каталог (1 услуга, 100⭐).

### 3. Приёмочный смоук
- `/start` → меню → «Услуги» → оплатить Stars → заполнить анкету (ФИО + дата ДД.ММ.ГГГГ)
  → прийти должен **PDF-отчёт**.
- «Мои расчёты» → отчёт открывается повторно.
- От админ-аккаунта: `/admin` (статистика), `/refund <charge_id>` (возврат звёзд).
