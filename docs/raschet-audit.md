# Аудит листа РАСЧЕТ — покрытие движком (2026-06-11)

Поячеечная сверка главного листа-отчёта `РАСЧЕТ` (то, что видит заказчик в своей
программе) с реализацией в боте. Метод: извлечение всех ячеек через openpyxl в
двух режимах — формулы (`data_only=False`) и кэш-значения (`data_only=True`),
классификация (подпись / активная формула / статика) и трассировка каждой
активной формулы до источника (`Matr`/`calc`/`Bio`/`текст`) и до выхода движка.

**Итог: все 102 активные формулы покрыты; 3 встроенных графика реализованы;
психоматрица полная (17/17).**

## Ключевые гочи аудита

- **Условные/пороговые тексты зашиты в формулы РАСЧЕТ, НЕ в листе «текст».**
  Доступ к деньгам (AN14), Жизненные силы (AN15), уровень сознания (C21), код
  тысячника (C19) — их не найти поиском по «текст», только через формулы ячеек
  (`data_only=False`). Реализованы хелперами в `core/render.py`.
- **Встроенные графики** достаются через `ws._charts[].series[].val.numRef.f`
  (ссылки на серии); их `title` — точные ярлыки для бота.
- **Нумерологический год помесячного графика идёт с февраля** (Фев→позиция 0,
  Янв→11); это НЕ Луна-Солнце-разница (`compute_monthly_moon_sun.value`).
- **Мёртвая ветка C21** `IF(AN8<AN8,…)` (всегда ложь) трактована как «иначе» →
  «наблюдатель» (политика проекта: каноническое намерение, не баг).
