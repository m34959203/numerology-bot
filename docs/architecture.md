## Обзор

```

│  Telegram  │ ◄─────────────────► │  bot/  (aiogram 3)        │

└────────────┘                     └────────────┬─────────────┘
```

## Принцип разделения

- **bot/** ничего не знает о нумерологии — собирает анкету, проводит оплату, зовёт `core` и форматирует ответ.
- **core/numerology/** — чистые функции (детерминированные, тестируемые без Telegram/БД). Каждый модуль = перенос конкретного листа Excel.
- **core/content/** — данные (тексты-трактовки), а не логика.

## Поток данных

1. `catalog` → пользователь выбирает услугу.
2. `payment` → invoice Telegram Stars → `pre_checkout` → `successful_payment` (запись в `payments` с `telegram_payment_charge_id`).
3. `survey` (FSM) → собирает ФИО + даты → запись в `survey_data`.
4. `report.build()` → дёргает нужные модули `numerology`, сопоставляет с `content`, собирает отчёт → `results`.
5. `results` → выдача (текст с разбивкой по 4096 / PDF), доступ через «Мои расчёты».

## Идемпотентность платежей

`telegram_payment_charge_id` уникален → повторный `successful_payment` не создаёт второй выдачи. Возврат — `refund_star_payment` админом, статус платежа `refunded`.

## Перенос Excel

Логика восстанавливается по листам целиком (не по ячейкам). Соответствие лист↔модуль — в [CLAUDE.md](../CLAUDE.md). Excel-функции воспроизводятся хелперами в чистом Python.
