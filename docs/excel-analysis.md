# Реверс-инжиниринг Excel — рабочие заметки

Источник истины: `docs/ПРОГРАММА МАТРИЦА на русском.xlsx` (50 листов).
Здесь — восстановленные алгоритмы по листам. Пополняется по этапам.

> **Ключевой приём.** В книге сохранены кэшированные значения для примера-входа
> **C12 = 01.01.2000** (ФИО и даты родителей пусты). Эти значения = готовые
> golden-эталоны для методик, зависящих только от даты (психоматрица, биоритмы,
> часть кодов). Читаются через `openpyxl(data_only=True)`.

## Лист РАСЧЕТ — вход и сборка отчёта

**Вход** (ячейки): `N9` — дата рождения; `N16 = TODAY()`. Метки ФИО/дат — в колонке `C`
(C6 Фамилия, C7 Имя, C8 Отчество, C9 дата рожд., C11 мать, C12 отец, **C14 девичья
фамилия — подпись по-казахски** «Тұрмысқа шыққанға дейінгі тегі» → интерфейс двуязычный).

**Выход** агрегируется ссылками на другие листы:
- блок «Вычисления» (AC/AN) ← `Matr` (код человека/жизни, фин-код, опасный возраст…);
- блок «Психоматрица» (K/O/S/W) ← `calc`;
- блок «Дни» (BD) ← `Bio` (благоприятные/критические/травмоопасные);
- блок «Прогноз на 5 лет» (B52+) ← `Matr` (персональное число года, циклы, Луна/Солнце).

## Лист calc — психоматрица ✅ реализовано (`core/numerology/psychomatrix.py`)

`C12 = РАСЧЕТ!N9`. Алгоритм квадрата Пифагора:

1. **Цифры даты** (D16:K16): день(2 цифры), месяц(2, добивка 0 при однозначном), год(4).
2. **Рабочие числа**: `E18` = сумма 8 цифр; `E19` = свод E18; `E20 = E18 − 2×(первая
   ненулевая цифра дня)` (спец-ветка при <0); `E21` = свод E20.
   Строка `U16 = concat(E18,E19,E20,E21)`, её цифры идут в пул.
3. **Пул** = цифры даты + цифры рабочих чисел. `COUNTIFS` вхождений 1..9 (нули не в счёт)
   = клетки квадрата (U19:W21).
4. **Качества** (РАСЧЕТ): повтор-цифры клеток (U24=count×digit) и суммы рядов/столбцов
   (целеустремлённость=c1+c4+c7, самооценка=c1+c2+c3, и т.д.). 0 → «нет».
5. **Скаляры**: число души `F1=DAY`; уровень сознания `C1=день+месяц+Σцифр года`;
   ЧЖП `J19=VLOOKUP(E19)` = свод `((n-1)%9)+1` (мастер-числа НЕ сохраняются!);
   код жизни `C14 = день×месяц×год`, добивка нулями до 6 знаков;
   код поведения `D13 = VLOOKUP(дата)` — таблица 13 «кодов» по диапазонам дат
   → выгружена в `core/content/data/behavior_codes.json` (ключ `MM-DD`).

Golden-эталон 01.01.2000 → `tests/fixtures/psychomatrix_2000_01_01.json` (тест зелёный).

## Лист Bio — биоритмы ✅ реализовано (`core/numerology/biorhythm.py`)

37677 строк = по дню от даты рождения (`B2=РАСЧЕТ!N9`, `B3=B2+1`, …).

**Фазовые аккумуляторы** (рекуррента, каждый день):
- физ: `I += 1.0131383374031213` (Bio!AF$3), перенос при `I>23` (−24);
- эмо: `J += 1.0201714790837135` (Bio!AC$15), перенос при `J>28` (−29);
- интел: `K += 1.0252235280781246` (Bio!AC$20), перенос при `K>33` (−34).
- Инкременты выведены из целочисленных констант листа; взяты как точные double.
- `F/G/H = ROUND(I/J/K)` (Excel ROUND — half-away-from-zero); сброс в 0 на следующий
  день после максимума цикла; при отрицательном аккумуляторе F/G/H = длина цикла.

**Зоны** (Bio!R/S/T): R(физ)=1 при 4≤F≤9, =2 при 17≤F≤21; S(эмо)=1 при 4≤G≤10,
=2 при 19≤G≤24; T(инт)=1 при 5≤H≤11, =2 при 23≤H≤28.

**Классификация дня** (Bio!U/V/W):
- благоприятный = R+S+T=3 (все зоны =1);
- критический = R+S+T=6 (все зоны =2);
- травмоопасный = 11.5 < I < 12 (физ-аккумулятор у пика).

РАСЧЕТ (BD6/BD9/BD12) выводит ближайшие такие дни вперёд от «сегодня» через
INDEX/MATCH-компакцию колонок U/V/W. **Верифицировано:** рекуррента+классификация
дают ровно даты книги (вход 01.01.2000, сегодня 27.07.2025): 7 благоприятных,
5 критических, 5 травмоопасных → `tests/fixtures/biorhythm_2000_01_01.json`.

**Открыто:** точные правила ВЫВОДА (число дат на категорию, длина окна) — у Excel
по категориям различаются (благоприятные считаются кластерами, критические
ограничены ~годовым окном). Сам движок от этого не зависит. См. OPEN_QUESTIONS.

## Лист Matr — ядро расчётов ✅ реверс-инжиниринг (golden 01.01.2000, нулевой допуск)

`Matr` (158×142, ~4715 формул). Все формулы ниже сверены с КЭШЕМ книги для входа
**01.01.2000**, «сегодня» = 27.07.2025 (`D9`). Сверка численно воспроизведена скриптом —
совпадение точное по всем перечисленным выходам.

### M.0 Входные ячейки Matr (тянут из РАСЧЕТ)

| Matr | Источник | Смысл | Кэш (пример) |
|------|----------|-------|--------------|
| `C3` | `РАСЧЕТ!N6` | Фамилия | пусто → 0 |
| `C4` | `РАСЧЕТ!N7` | Имя | пусто → 0 |
| `C5` | `РАСЧЕТ!N8` | Отчество | пусто → 0 |
| `C6` | `РАСЧЕТ!N14` | Девичья фамилия | пусто → 0 |
| `C7` | `РАСЧЕТ!N9` | **Дата рождения** | 01.01.2000 |
| `P4` | `РАСЧЕТ!N11` | Дата рождения матери | пусто |
| `P5` | `РАСЧЕТ!N12` | Дата рождения отца | пусто |
| `D9` | `РАСЧЕТ!N16` | Сегодня (`TODAY()`) | 27.07.2025 |

В примере заполнена только `C7`. ФИО и даты родителей пусты → все ФИО-зависимые
выходы = 0 (см. M.7).

### M.1 Разложение даты на цифры (строка 7, AN7:AU7)

```
AN7 = MID(DAY,1,1)            # 1-я цифра дня
AO7 = MID(DAY,2,1) or 0       # 2-я цифра дня (0, если день однозначный)
AP7 = MID(MONTH,1,1)          # 1-я цифра месяца
AQ7 = MID(MONTH,2,1) or 0     # 2-я цифра месяца
AR7..AU7 = 4 цифры года
```
ВНИМАНИЕ: день/месяц НЕ добиваются ведущим нулём (в отличие от листа `calc`).
Для 01.01.2000 → AN7..AU7 = 1,0,1,0,2,0,0,0.

### M.2 Код человека / духовный уровень (BD7, BE7, BF7, BG7) — выходы AN9/AN10/AN12

```
BD7 = AN7+AO7+AP7+AQ7+AR7+AS7+AT7+AU7      # сумма 8 цифр даты = «духовный уровень»
BE7 = reduce2(BD7)                          # свод до 1-2 цифр: d1 + (d2 или 0)
BF7 = |BD7 − (AN7*2 если AN7>0 иначе AO7*2)| # «парное/теневое» число
BG7 = reduce2(BF7)
```
- `РАСЧЕТ!AN9 = BD7` = духовный уровень = **4**
- `РАСЧЕТ!AN10 = CONCAT(BD7,BE7)` = код человека = **"44"**
- `C8 = CONCATENATE(BD7,BE7," ",BF7,BG7)` = **"44 22"** (полный код)
- `AN12` счастливые числа = `BE7` (=4) и `BG7` (=2)

`reduce2(n)` = `int(str(n)[0]) + (int(str(n)[1]) if len>1 else 0)` — свод двузначного
в одну сумму цифр (НЕ итеративный, один проход; мастер-числа не выделяются).

### M.3 Код жизни (C9, BI7) — выход AN11

```
C9  = DAY * MONTH * YEAR                    # = 2000 (код жизни «сырой»)
BI7 = pad(C9)                               # дополнение нулями до фикс. длины
  если C9 < 10000:  результат = C9*100, затем если <100000: *10  → C9*1000
  иначе если <100000: C9*10
  иначе: C9
```
Для 2000: 2000<10000 → ветвь даёт **200000**. `РАСЧЕТ!AN11 = BI7 = 200000`.
(Формула BI7 написана криво через вложенные IF, но кэш = 200000; воспроизведено точно.)

### M.4 Финансовый код удачи (D8) — выход AN13

Свод-цепочка `red(n) = n if n<=9 else reduce2(n)`, применяется до стабилизации
(в книге — 2-3 уровня AO9→AO10→AO11):

```
AO9 = AN7+AO7              # сумма цифр дня      → AO11 = red·red(AO9)
AQ9 = AP7+AQ7             # сумма цифр месяца   → AQ11
AU9 = AR7+AS7+AT7+AU7     # сумма цифр года     → AU11
AV9 = AO11+AQ11+AU11      → AV11
D8  = CONCAT(AO11, AQ11, AU11, AV11)
```
Для примера: AO11=1, AQ11=1, AU11=2, AV11=4 → `D8 = "1124"`. ✅

### M.5 Доступ к деньгам / Жизненные силы (C11, C12) — выходы AN14/AN15

```
BO7 = CONCAT(MONTH, YEAR) * DAY * 100        # = 1200000
C11 = сумма цифр BO7                          # = 3  «доступ к деньгам 27»
BO8 = CONCAT(DAY, MONTH) * YEAR * 100         # = 2200000
C12 = сумма цифр BO8                          # = 4  «жизненные силы 27»
```
(C11/C12 = `BP7+…+BV7` / `BP8+…+BV8`, где BPx..BVx = посимвольные цифры BO7/BO8.) ✅

### M.6 Жизненная задача (C10) — выход AN8

```
C10 = COUNTBLANK(AN6:BC6)
```
`AN6..BC6 = IF(VALUE(цифра)<1, 1, "")` — помечают НУЛЕВЫЕ позиции единицей, ненулевые —
пустой строкой. Пул из 16 позиций: 8 цифр даты (AN7:AU7) + 8 цифр кода человека
(AV7:BC7 = цифры BJ7=`CONCAT(BD7,BE7,BF7,BG7)`="4422", добитого нулями).
`COUNTBLANK` = число НЕнулевых позиций = **7**. ✅

### M.7 Карма имени / Число имени (D3, D7) — выход AN17. ЗАВИСИТ ОТ ФИО

Транслитерация кириллицы в цифры (правый блок BJ1:DB2):
- `BJ1:DB1` = алфавит А..Я (33 буквы); `BJ2:DB2` = цифры 1-9 циклично (А=1,Б=2,…,И=9,Й=1,…).
- Для каждой буквы имени: `MATCH(буква, BJ1:CP1)` → позиция → свод `red(pos)`.
  ⚠️ MATCH возвращает позицию 1..33, далее свод `red()` доводит её до 1-9 — но это НЕ
  то же, что чтение цифры из BJ2 (там цикл по 9). На пустом ФИО проверить нельзя
  (см. открытые вопросы).
- `D3 = DP1` число фамилии; `D4/D5/D6` — имени/отчества/девичьей; всё через DP/DQ/DR-цепочку
  сводов сумм по строкам DS..EL.
- `D7 = DR12 = SUM(DP1:DP9)` = карма имени; `E7` помечает «карма имени», если ∈{13,14,16,19}.
- `РАСЧЕТ!AN17 = D7`. **В примере ФИО пусто → DS4="" → MATCH=#N/A → IFERROR=0 → D7=0.** ✅
- `I3 = G3 + H3` (код тысячника, `C19`): `G3=IF(BE7>9,BD7,0)`, `H3=IF(BG7>9,BF7,0)`.
  Для примера BE7=4≤9, BG7=2≤9 → `I3 = 0`. ✅ (Срабатывает только при двузначных сводах.)

### M.8 Полных лет (AF34 / I29) — выход AN6

```
X29  = J29 − C7                              # дней прожито (J29 = текущий год прогноза)
Z34  = 1 если (мес_сегодня−мес_рожд>0) иначе (1 если день_сег−день_рожд>=0 иначе 0)
AC34 = 1 если мес_сегодня>=мес_рожд иначе 0
AE34 = 1 если год_сег>=год_рожд иначе 0
AF29 = YEAR(сегодня) − YEAR(рожд)            # «грубый» возраст = 2025−2000 = 25
AF34 = AF29−1 если (AE34+AC34+Z34)<3 иначе AF29
```
Т.е. полный возраст с поправкой −1, если ДР в этом году ещё не наступил. Для 01.01 (ДР в
январе, сегодня июль) поправки нет → **25**. `Matr!I29 = AF34 = 25`. ✅
Эквивалент в Python: `years = today.year − bd.year − ((today.month,today.day) < (bd.month,bd.day))`.

### M.9 Опасный возраст (F8) — выход AN16

```
F8 = MATCH(0, AM60:AM158, 0)
AMr = IF(BI_r >= AM29, BM_r, "")            # BM_r = Солнце года r; AM29 = текущий год прогноза (2025)
```
Т.е. среди годов, начиная с текущего (BI≥2025), ищется ПЕРВЫЙ, где «Солнце» = 0.
`MATCH` отдаёт его 1-based позицию в диапазоне AM60:AM158 → **25** (= возраст наступления). ✅
Семантика «позиция-как-возраст» работает, т.к. строка r соответствует возрасту `BK_r = r−59`.

### M.10 Таблица годов BI59:BN158 (годы рождения+0..99 → 2000..2099)

Сетка из 100 строк (59..158), одна строка = один год жизни:

| Кол. | Формула | Смысл |
|------|---------|-------|
| `BI` | `BI59=YEAR(C7)`, далее `+1` | Календарный год (2000..2099) |
| `BK` | 0,1,2,…,99 (константы) | Возраст |
| `BH`=`BX` | `IFERROR(BW,BT)` | «Общее число года» = свод цифр года (BS=Σцифр→BT 1-я цифра, BW = свод; 10→1) |
| `BJ` | свод `(AN7+AO7+AP7+AQ7+BH_r)` до 1 цифры (10→1) | **Персональное число года** |
| `BL` | `BZ+CA` | **Луна** года (см. ниже) |
| `BM` | `CB+CC` | **Солнце** года |
| `BN` | `BM−BL` | Год на +/− (баланс) |
| `BE` | интерполяция энергопотенциала (M.11) | Энергетический потенциал |
| `BF` | `CR$` сегмента | Знак тренда энергии (для стрелки ↗/↘) |
| `BG` | `CH$` сегмента | Судьбоносные события (anchor-цифра) |

**Луна/Солнце (BL/BM)** через делёж кода жизни на возраст:
```
BY_r = ROUNDDOWN(BI7 / возраст, 0)          # BI7 = 200000 (код жизни padded)
BZ,CA,CB,CC = MID(BY,1..4,1)                 # первые 4 цифры
Луна(BL)  = BZ + CA      (две старшие цифры)
Солнце(BM)= CB + CC      (3-я и 4-я цифры)
возраст 0 → Луна=Солнце=0
```
Проверено точно: возраст 25→(Луна 8, Солнце 0); 26→(13,11); 27→(11,7); 28→(8,6);
29→(14,15); 30→(12,12) — совпадает с `F29:G33`. ✅

### M.11 Энергетический потенциал — таблица BE59:BE158 + блок CG73:CZ88

Это **кусочно-линейная интерполяция** между «опорными» значениями каждые 7 лет.

1. Цифры кода жизни (`C9`): `BP9..BU9 = MID(C9,1..6)` → для 2000: 2,0,0,0,(пусто),(пусто).
2. Опорный массив `CH73:CH88` (16 точек на возрасты CG = 0,7,14,…,105) строится
   ротацией этих цифр по фиксированной схеме `IF(CN72="",…)` (число значимых цифр кода
   жизни определяет ветку). Для примера → `CH = [2,0,0,0, 2,0,0,0, 2,0,0,0, 2,0,0,0]`.
3. Шаг сегмента: `CS_seg = (следующий_anchor − текущий_anchor) / 7` (`CR=разница`, делёж на 7).
4. `BE_r` = линейная интерполяция: `anchor + CS*offset` для каждого года внутри 7-летнего сегмента.

Проверено точно: BE для возрастов 0..7 = 2.0, 1.7143, 1.4286, 1.1429, 0.8571,
0.5714, 0.2857, 0.0 — совпадает с кэшем BE59..BE63 (2, 1.7143, 1.4286, 1.1429, 0.8571). ✅
`BE59:BE158` = выход прогноза энергопотенциала по всем 100 годам жизни.

### M.12 Прогноз на 5 лет (строки 29:33, выход РАСЧЕТ C29:J33)

Каждая строка прогноза r (29..33) = один будущий год:
```
J29 = AH28 = сегодня (27.07.2025); J30 = J29+366; J31=J30+365; … (грубый +год)
I29..I33 = AF34, AF35=AF34+1, …  → возраст 25,26,27,28,29
AM_r = YEAR(J_r)                 # календарный год прогноза
AL_{r+5} = MATCH(год, BI59:158)  # индекс года в таблице
AI_r = MATCH(возраст, BK59:158)  # индекс возраста в таблице

B_r (стрелка) = "↗" если INDEX(BF59:158, AI_r) > 0 иначе "↘"   # тренд энергии
C_r (перс.число года) = INDEX(BJ59:158, AL_{r+5})              # 2,3,4,5,6
F_r (Луна)   = INDEX(BL59:158, AI_r)                            # 8,13,11,8,14
G_r (Солнце) = INDEX(BM59:158, AI_r)                            # 0,11,7,6,15
H_r (год +/−)= G_r − F_r
D_r («судьбоносные события») = "+" если MATCH(год, BI9:48) найден, иначе "−"
E_r = если D_r="+": CONCAT(IF(AG_r,"добро ","зло "), AH_r)
   AG_r = ISODD(AH_r);  AH_r = INDEX(BG59:158, AI_r)            # anchor-событие
```
Проверено: персональные числа года = 2,3,4,5,6; стрелки ↗↗↗↗↘ — совпадает с кэшем. ✅

### M.13 Индексы AJ28:AJ34 (для прогноза энергопотенциала)

```
AJ28 = AJ29 − 1
AJ29 = AF34            # = полный возраст (25)
AJ30 = AF35, AJ31 = AF36, AJ32 = AF37, AJ33 = AF38   # 26,27,28,29
AJ34 = AJ33 + 1        # = 30
```
То есть `AJ28:AJ34` = последовательность возрастов 24..30 (центр = текущий возраст 25 ±).
Используются как аргумент `AI_r = MATCH(AJ_r, BK59:158)` для выборки из годовой таблицы
(энергия BF/BE, Луна/Солнце). `AI28 = AI29−1`, `AI34 = AI33+1` — расширяют окно на ±1 год
для построения тренда стрелки (B29:B34 — 6 значений, на год шире 5-летнего прогноза).

### M.14 12-летний цикл и кармические AF4/AF5

```
AF4 = "+" если AE4 чётное иначе "−"        # AE4 — год (?) ; в примере "−"
AG4 = BD7 + AE4                             # = 7 (духовный уровень + …)
AF5 = "+" если AE5 > AD5 иначе "−"          # в примере "−"
AG5 = AC5 + AB5                             # в примере #VALUE! (нет входов)
```
Блок 12-летнего цикла (E28 «12-летний цикл») собирается в E29:E33 как
`CONCAT(IF(AG_r,"добро ","зло "), AH_r)` (см. M.12, поле E). РАСЧЕТ тянет

### M.16 Открытые / неоднозначные места (Matr)

1. **Транслитерация ФИО (M.7).** На пустом ФИО невозможно проверить, использует ли книга
   `MATCH(позиция 1..33)→red()` или цифру из BJ2 (цикл 1-9). Это РАЗНЫЕ результаты для
   букв после «И» (поз.10+). Нужен контрольный пример с заполненным ФИО от заказчика
   ДО реализации `число имени`/`карма имени`. **Не угадывать.**
2. **Мастер-числа.** `reduce2`/`red` сводят однопроходно и НЕ выделяют 11/22/33 как мастер
   (BJ59-формула явно гасит только 10→1). Для согласованности с листом `calc` (где ЧЖП
   тоже не хранит мастера) — поведение единое, но подтвердить на ФИО-кейсе.
3. **`BI7` (M.3).** Формула паддинга кода жизни запутанная (двойные вложенные IF). Для
   значений с другим числом цифр (напр. ДР с произведением d*m*y >100000) ветка иная —
   проверить на кейсе, где d*m*y ≥ 100000 (крупные дни/месяцы поздних лет).
4. **AF4/AF5 + AB:AE строк 4-5 / AG5=#VALUE!** Кармический блок частично не заполняется на
   чистой дате. Нужно понять, при каких входах он активируется (вероятно зависит от ФИО
   или дат родителей P4/P5) и что именно выводит РАСЧЕТ из AF4/AF5.
5. **«Судьбоносные события» D_r/E_r (M.12).** `MATCH(год, BI9:48)` ссылается на таблицу
   BI9:BK48 (40 строк, не разобрана детально) — это список «судьбоносных годов».
   Её генерацию (что попадает в BI9:48) надо восстановить отдельно перед прогнозом событий.
6. **«Год на +/−» H_r** = Солнце−Луна; смысловая трактовка (что показывает знак) — вопрос
   к контенту, не к формуле.
7. **E8 «СОЛНЦЕ 0» / E10 «Код долгожителя»** — служебные флаги (E8 ✅ срабатывает при
   наличии года с Солнцем=0; E10=#VALUE! т.к. M10 не вычислен) — уточнить, выводятся ли в отчёт.

## Блок «Луна и Солнце по месяцам» + ЧПМ ✅ реверс-инжиниринг (golden 01.01.2000)

Листы: `луна и солнце` (зеркало входа), `Calculation луна и солнце` (движок),
`Lists луна и солнце` (справочники), `ЧПМ`. **Потребители — скрытые render-листы
`35` (помесячная Луна/Солнце) и `36` (ЧПГ/ЧПМ/ЧПД).** В РАСЧЕТ прямых ссылок нет —
это отдельные разделы отчёта. **Имя НЕ участвует** (таблица `Alfabet` присутствует, но
ни одна формула к ней не обращается); всё считается от даты рождения и даты отчёта.

### LS.1 Помесячная Луна/Солнце (`Calculation луна и солнце`)
```
LifeCode = pad6(day2·month2·year)                 # 01.01.2000 → 200000
YearNumber = digital_root(Σ цифр year(отчёта))    # 2025 → 9   (мастер не хранит)
# изменённый код: каждая из 6 цифр LifeCode + YearNumber, свод → [2,9,9,9,9,9]
age = INT((отчёт − рождение)/365)                 # 25
B10 = LifeCode/age = 8000.0 ; Moon=цифра1+2, Sun=цифра3+4 ; Result = Sun−Moon  # −8
# дважды изменённый код B26 = свод(изм.код + Result) по 6 позициям → 611111
divisors[12] = Lists!K2:K13 = [7,9,28,10,8,16,11,14,17,25,2,19]
для месяца m: C = B26/divisor[m]; первые 4 ЦИФРЫ строки C → D,E,F,G
   Sun_m=F+G, Moon_m=D+E ; J = (Sun_m−Moon_m) если Result≠0 иначе −ABS(...)
   текст = VLOOKUP(J, Lists!G2:H38)               # ключи −18..18
```
Сверка J по 12 месяцам: `-12,-4,7,-5,-2,-1,0,4,5,2,7,2` — все совпали с кэшем.

### LS.2 ЧПГ/ЧПМ/ЧПД (`ЧПМ`)
```
reduce(n)=((n-1)%9)+1
ЧПГ = reduce( digital_root(Σцифр year(отчёта)) + reduce(day+month) )   # 2
ЧПМ = reduce( ЧПГ + month(отчёта) )                                    # 9
ЧПД = reduce( ЧПМ + day(отчёта) )                                      # 9
ключ комбинации = f"{ЧПМ} − {ЧПГ}"   # разделитель U+2212, НЕ дефис!
```
Тексты (лист ЧПМ → render-лист 36): `N2:O10` ЧПМ 1–9; `W2:X10` ЧПД 1–9;
`Q2:R82`/`T2:U82` — 81 комбинация «ЧПМ − ЧПГ» (заголовок + развёрнутый текст).

**Гочи:** (1) извлечение «первых 4 цифр» из дробного частного чувствительно к формату
float — закрепить тестом; (2) `G5=6` в движке — хардкод-константа (семантика «прошёл ли
ДР»), для воспроизведения достаточно `L7=year(отчёта)`; (3) есть мёртвые ветки (`#N/A`,
`_xlfn.IFS`, B26-дубль) — игнорировать; (4) нужен доп.эталон на «толстом» годе (Σцифр>9).
