## Сверка методик с формулами Excel (2026-06-10)

Проверено по формулам (data_only=False), не по догадкам:
- ✅ **ИТОГ года** (листы 29–33 `D6 = VLOOKUP(Matr!H29..33, текст!H363:I401, 2)`).
  `Matr!H = Солнце − Луна` = наш `year_value`. На эталоне Ерофеева 20.02.1990
  (ref 06.06.2026) книга и движок дают **[-2, 3, 11, 3, -1]** — точное совпадение.
  Лукап `year_total` по `year_value` — верен.
- ✅ **График кода жизни — РЕВЕРС ЗАВЕРШЁН (2026-06-10).** Лист 19
  `D36 = VLOOKUP(F37, текст!B213:C222, 2)`, `F37 = INDEX(F21:K21, MAX(F39:F51))`. Разобрано:
  `F21:K21` = 6 цифр `AR32 = AN11 = код жизни` (796000); сетка `F22:K34` — возрасты 0–77 по 6
  в ряд; `MAX(F39:F51) = (age%6)+1`. Итог: **активная цифра = `код_жизни[age % 6]`** →
  текст 0–9. Эталон: код 796000, возраст 36 → цифра 7 (= F37). `matrix.life_code_graph_digit`,
  подключено per-год в прогнозе. Первое (неверное) подключение по `rebirth_cycle` исправлено.
- ✅ **Кодировки «до/после 35» — СДЕЛАНО (2026-06-10).** Реверс `calc!E18..E21` сошёлся:
  `E18 = Σцифр даты = spiritual`, `E19 = свод(E18) = be7` → `1-код = {spiritual}{be7}` =
  наш `human_code` (=235); `E20 = abs(spiritual − 2×первая_цифра) = bf7`, `E21 = свод(E20)
  = bg7` → `2-код = {bf7}{bg7}` (=1910). Оба ключа есть в `human_code.json`. Добавлено
  `codes.second_code`; `report` тянет описания обоих; render/pdf выводят блок «Кодировка
  жизни»: «До 35 лет — код 235 …» / «После 35 лет — код 1910 …». Эталон Ерофеева — точно.
- ℹ️ Книга в годовом прогнозе выводит ТОЛЬКО текст ИТОГ; таблицы Луна (B363:C381) и
  Солнце (E363:F382) — источники. «Нулевой период» (Солнце=0, `F382`) книга отдельным
  per-год лукапом не выводит — мы показываем его как справку **по требованию заказчика**.
