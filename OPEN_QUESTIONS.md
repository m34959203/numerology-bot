# Открытые вопросы

> **Рамка:** заказчик дал Excel-таблицу как ЭТАЛОН своей программы. Задача — перенести
> её в Telegram-бота 1:1. Поэтому поведение расчётов берём ИЗ таблицы, а не спрашиваем.
> Сюда попадает только то, что таблица объективно не определяет.
> Статус: 🔴 ждём ответа · 🟡 есть гипотеза/обходной путь · 🟢 закрыто по таблице.

## Закрыто по таблице (бывшие вопросы)

- 🟢 **Excel-источник** получен: `docs/ПРОГРАММА МАТРИЦА на русском.xlsx`.
- 🟢 **Обязательные поля.** Для основного отчёта обязательна ТОЛЬКО дата рождения.
  Имя + имя отца нужны лишь для разделов «Финансовый прогноз» и «ЧПМ/ЧПД» (подпись BD44).
  Даты матери/отца и девичья фамилия на листе РАСЧЕТ в расчётах не используются (поля-задел).
- 🟢 **Сведение чисел.** Везде свод по mod-9 `((n-1)%9)+1`, мастер-числа 11/22/33 НЕ
  сохраняются (calc, Matr, ЧПМ — единообразно).
- 🟢 **Горизонт биоритмов.** Это presentation таблицы: воспроизводим то, что показывает
  РАСЧЕТ (в примере 7 благоприятных / 5 критических / 5 травмоопасных). Движок точен.
- 🟢 **Язык.** Контент трактовок — русский; казахский в книге только в подписях полей.
  Бот — русскоязычный (kk при необходимости позже, отдельной задачей).

## Реально нужно от заказчика

### Расчёты (нельзя проверить без данных)
- 🔴 **Контрольный пример с ЗАПОЛНЕННЫМ ФИО.** В кэше книги ФИО пусто → расчёты от имени
  («число имени», «карма имени», финансовый прогноз) не на чем верифицировать. Нужен
  1–2 эталона `(ФИО + дата) → результат` из программы заказчика. Без этого «число имени»
  не реализуем точно (транслитерация букв после «И» неоднозначна — см. excel-analysis M.16).
- 🟡 **Баги исходной книги** (воспроизводить как есть или чинить?): `C21` (мёртвая ветка
  духовного уровня), `AN17` карма имени (`OR(...)` всегда TRUE), `BG20` #VALUE! (дробный
  MID). По умолчанию реализуем КАНОНИЧЕСКОЕ намерение (см. excel-analysis R.1/R.2), не баг.
